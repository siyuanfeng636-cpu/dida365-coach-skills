# Dida Coach Skill 设计文档

**日期**: 2026-03-17
**版本**: v1.0
**状态**: 待实现

---

## 1. 概述

### 1.1 目标
创建一个结合滴答清单 MCP 的 Claude Code Skill，作为用户的"任务教练"，实现：
- 想法落地督促（目标拆解为周期性/阶段性任务）
- 解决行动力不足（任务拆分 + 时间盒子）
- 情感伴侣支持（激励 + 未完成支撑方案）
- 零散任务闭环管理
- 定期复盘（日复盘、周复盘）

### 1.2 目标用户
- 有自我提升需求但缺乏执行力的人
- 任务管理混乱、容易拖延的人
- 需要外部 accountability（问责）的人

## 3. 核心功能设计

### 3.1 滴答清单 MCP 接入

**首次启动检测流程：**

```yaml
setup:
  required_mcp:
    - name: dida365
      url: https://mcp.dida365.com
      description: 滴答清单 MCP，用于任务读写

  setup_flow:
    1. 检测 ~/.claude/mcp.json 中是否已配置 dida365
    2. 若未配置，显示一键配置命令
    3. 引导用户运行 /mcp 完成 OAuth 授权
    4. 验证连接成功后，进入主功能
```

**首次对话模板：**
```
👋 欢迎使用 Dida Coach！

检测到您尚未连接滴答清单，请先执行以下步骤：

1. 运行：claude mcp add --transport http dida365 https://mcp.dida365.com
2. 然后运行：/mcp
3. 按提示完成滴答清单授权

完成后我会自动检测连接。需要我帮您检查状态吗？
```

### 3.2 任务拆解流程

```
用户输入："我想提高英语口语"

AI 处理流程：
1. 询问当前水平（多选）
   A. 零基础  B. 能简单对话  C. 流利但想更地道

2. 询问时间投入
   "每周能投入多少时间？"

3. 询问具体场景
   "主要用于？工作/留学/旅行/兴趣"

4. 生成阶段性计划
   "基于您的情况，建议 3 个月计划：
    第1月：发音基础（每日 15 分钟）
    第2月：日常对话（每日 30 分钟）
    第3月：话题拓展（每日 30 分钟）

    需要我创建到滴答清单吗？"

5. 用户确认后，调用 MCP 创建任务：
   - 清单：自我提升
   - 任务：英语口语-第1月-发音基础
   - 子任务：Day1-30 每日练习
   - 标签：#习惯养成 #语言学习
   - 优先级：!1
```

### 3.3 时间盒子设计

#### 3.3.1 工作法配置

```yaml
# 默认工作法配置
work_method:
  default: flexible_pomodoro

  methods:
    flexible_pomodoro:
      name: "灵活番茄（默认）"
      focus: 30        # 专注时长（分钟）
      short_break: 5   # 短休息（分钟）
      long_break: 15   # 长休息（分钟）
      boxes_before_long: 4  # 几个盒子后长休息
      max_consecutive: 4    # 连续上限

    classic_pomodoro:
      name: "经典番茄"
      focus: 25
      short_break: 5
      long_break: 15
      boxes_before_long: 4
      max_consecutive: 4

    long_pomodoro:
      name: "长番茄（深度工作）"
      focus: 50
      short_break: 10
      long_break: 30
      boxes_before_long: 2
      max_consecutive: 2

    ultradian:
      name: "90分钟深度"
      focus: 90
      short_break: 0   # 无短休息
      long_break: 20
      boxes_before_long: 1
      max_consecutive: 1
```

#### 3.3.2 智能工作法推荐

| 任务类型 | 推荐工作法 | 理由 |
|---------|-----------|------|
| 编程/写作 | 长番茄 (50+10) | 减少上下文切换，保持心流 |
| 邮件/沟通 | 灵活番茄 (30+5) | 快速处理，适合碎片化 |
| 创意构思 | 90分钟深度 | 需要长时间进入心流 |
| 学习/阅读 | 灵活番茄 (30+5) | 便于吸收，及时休息 |
| 会议/协作 | 灵活番茄 (30+5) | 会议通常以 30 分钟为单位 |

#### 3.3.3 时间盒子创建流程

```
用户："我要写一份项目汇报"

AI：
1. 任务分析
   "项目汇报大概涉及哪些方面？"
   A. 进度总结  B. 问题分析  C. 下阶段计划  D. 全都包含

2. 根据任务类型推荐工作法
   "写报告属于深度工作，建议用长番茄模式（50分钟），
    或标准 30 分钟盒子。你想用哪种？
    A. 长番茄（50分钟，适合深度）
    B. 灵活番茄（30分钟，默认）"

3. 估算时间盒数量
   "建议拆分为 4 个时间盒子（2 小时）：

   📦 盒子 1 (09:00-09:30) | 成果：进度框架 ✓
      └─ 完成项目背景 + 当前进度概述

   📦 盒子 2 (09:35-10:05) | 成果：问题清单 ✓
      └─ 梳理遇到的问题及原因

   📦 盒子 3 (10:10-10:40) | 成果：解决方案 ✓
      └─ 针对问题给出解决方案

   📦 盒子 4 (10:45-11:15) | 成果：完整报告 ✓
      └─ 整合 + 下阶段计划 + 格式美化

   🔄 长休息 11:15-11:30

   需要调整吗？你可以：
   • 说'改成 45 分钟'调整时长
   • 说'下午开始'改时间
   • 说'删除盒子 3'简化任务"

4. 用户确认/调整后
   "好的，已创建到滴答清单！
    我会在每个盒子结束时提醒你确认成果。"

5. 设置检查点提醒（Cron）
   09:30, 09:35, 10:05, 10:10, 10:40, 10:45, 11:15
```

#### 3.3.4 检查点确认流程

```
⏰ 时间到！盒子 1 结束 (09:30)

"盒子 1 时间到！请确认成果：

📋 应完成：项目背景 + 当前进度概述

✅ 实际完成度？
A. 100% - 按计划完成，可以进入下一盒
B. 80% - 基本完成，小尾巴下一盒补
C. 50% - 需要延长 10 分钟
D. 没做完 - 遇到什么问题？

选择后：
A/B → "太棒了！休息 5 分钟，09:35 开始下一盒 ☕"
C → "好的，延长到 09:40，然后进入休息"
D → "聊聊遇到什么问题了？是任务太难还是分心？"
```

#### 3.3.5 改时间功能

用户可随时说：
- `"把盒子 2 改到下午 2 点"` → 单独调整
- `"所有盒子延后 1 小时"` → 批量调整
- `"盒子 3 改到 45 分钟"` → 调整时长
- `"今天状态不好，全部取消"` → 取消并记录原因
- `"切换长番茄模式"` → 改变工作法

### 3.4 复盘功能设计

**触发方式：**
- 用户主动："复盘今天"

```
• [任务名] - 原因：[用户输入/推测]

💡 明日建议
• 基于数据的时间安排建议
• 可优化的重复任务提醒
```

**触发方式：**
- 用户主动："复盘这周"

```
📈 本周统计

• 平均每日完成：X 个任务
• 最 productive 的一天：周X
• 拖延最多的类型：[类型]

🏆 本周成就
• 连续完成天数：X 天
• 值得庆祝的：[具体任务]

🔍 深度分析
• 未完成原因分布
• 可自动化任务识别
• 工作法效果对比

📋 下周计划建议
• 基于本周数据的优化建议
```

#### 3.5.2 闭环反馈机制

```
任务生命周期：

创建任务 ──► 设置检查点 ──► 到期询问 ──► 完成/未完成
     │                │              │           │
     └────────────────┴──────────────┘           │
                    │                            │
                    └──────── 记录原因/调整计划 ◄─┘

未完成时的支撑方案：
1. 询问原因（太难/没动力/分心/其他）
2. 根据原因提供支持：
   - 太难 → 进一步拆解任务
   - 没动力 → 联系目标意义、降低难度
   - 分心 → 建议环境调整、番茄工作法
   - 其他 → 记录并调整期望
3. 重新约定检查点
4. 追踪改进情况
```

## 6. 配置示例

```yaml
# ~/.claude/skills/dida-coach/config.yaml

# 用户文风设定
personality:
  preset: "warm_encouraging"
  # 或自定义
  # custom:
  #   prompt: "你是一位..."

# 工作法偏好
work_method:
  default: "flexible_pomodoro"
  auto_recommend: true  # 根据任务类型智能推荐

# 提醒时间
reminders:
  daily_plan: "09:00"
  daily_review: "21:00"
  weekly_review: "Sunday 20:00"
  enabled: true

# 复盘偏好
review:
  dimensions:
    - completion_rate      # 完成率
    - time_distribution    # 时间分布
    - task_type_analysis   # 任务类型分析
    - missed_reason        # 未完成原因
    - suggestions          # 建议
    - highlights           # 成就亮点
    - procrastination_time # 拖延时间段
    - automation_candidates # 可自动化任务

# 闭环设置
closure:
  check_in_enabled: true
  missed_task_follow_up: true  # 未完成时主动跟进
  max_reschedule_times: 3      # 最多改期次数
```

## 8. 实现注意事项

### 8.1 与 MCP 的错误处理

- MCP 未连接：友好提示配置步骤
- 授权过期：引导重新授权
- 网络问题：本地缓存 + 重试机制
- 任务冲突：提示用户手动解决

### 8.2 隐私考虑

- 所有数据存储在用户的滴答清单账户
- 本地仅存储配置文件
- 复盘分析在本地完成，不上传

### 8.3 扩展性

- 支持后续添加其他 MCP（如 Notion、Todoist）
- 文风预设可扩展
- 工作法可自定义
